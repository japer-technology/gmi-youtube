# Aspiration

## I Love YouTube

My subscriptions feed is fabulous. It is a personally curated stream of creators, ideas, and content that reflects exactly what I care about. No algorithm fully controls it — I chose every channel, and the result is a feed that feels like mine. This project exists because that experience deserves to go further.

## The Vision

I want to turn this GitHub repository into a **personal TV station** powered by my YouTube account.

### Inside GitHub — Process YouTube into Resources

Using the **YouTube Data API** (authenticated via a `YOUTUBE_API_KEY` repository secret), GitHub Actions workflows will:

- **Pull my subscriptions** — Retrieve the channels I subscribe to and the videos they publish.
- **Pull my playlists** — Access my curated playlists as programmable collections of content.
- **Pull my liked videos** — Surface the videos I have explicitly endorsed.
- **Catalogue and index** — Transform raw API responses into structured data files (JSON, Markdown, or both) committed to this repository, making my YouTube activity a versioned, searchable, first-class dataset.
- **Refresh on a schedule** — Run periodically so the repository always reflects my current YouTube state without manual intervention.

The repository becomes the **backend** — a place where YouTube data is fetched, processed, normalized, and stored as static resources, all within GitHub's infrastructure.

### Outside GitHub — A GitHub Page References YouTube

The **GitHub Pages** site published from this repository becomes the **frontend** — my personal TV station interface. It will:

- **Present my content** — Display my subscriptions, playlists, and liked videos in a browsable, watchable format.
- **Link directly to YouTube** — Every video reference on the page is a URL back to YouTube. No re-hosting, no embedding workarounds — just clean links (or embedded players) that respect YouTube's platform.
- **Be configurable** — I decide what shows up, how it is organized, and what the experience feels like. Themes, filters, schedules, featured channels — all driven by the data in this repo.
- **Be static and fast** — As a GitHub Pages site, it requires zero servers, zero databases, and zero cost beyond what GitHub already provides.

## The Boundary

The architecture draws a clear line:

| Boundary | Role | What lives here |
|---|---|---|
| **Inside GitHub** (Actions, repo files) | Data processing | API calls, data transformation, JSON/Markdown resources, scheduled workflows |
| **Outside GitHub** (GitHub Pages) | Presentation | Static HTML/CSS/JS that reads the processed data and links back to YouTube URLs |

YouTube is the **source**. This repository is the **processor**. GitHub Pages is the **screen**.

## Why This Matters

- **Ownership** — My YouTube taste, expressed as data I control in a repo I own.
- **Transparency** — Every fetch, every transformation, every page update is a git commit I can inspect.
- **Composability** — Once my YouTube data is structured files in a repo, I can build anything on top of it: RSS feeds, dashboards, recommendations, cross-references with other data sources.
- **Simplicity** — No external services, no paid hosting, no complex deployments. Just GitHub Actions, GitHub Pages, and the YouTube API.
- **Longevity** — If YouTube changes, I adapt the processing layer. If GitHub Pages evolves, I adapt the presentation layer. The data in between is mine, versioned, and portable.

## The Aspiration, Deeply

This is not just a tool. It is a statement about how content consumption should work.

Today, platforms give us feeds but own the experience. We watch what they surface, in the order they choose, through the interface they designed. We have no way to remix, restructure, or repurpose our own viewing habits outside the walls of their app.

This project aspires to change that — at least for one person, for one platform. I want to take my YouTube subscriptions, the single best content feed I have, and project them through an interface I fully control. I want the data pipeline to be auditable. I want the presentation to be mine to shape. I want every piece of it to live in a repository where I can see exactly what happened and why.

**The aspiration is a personal TV station where GitHub is the broadcast infrastructure and YouTube is the content library.**

Every subscription is a channel. Every playlist is a programme schedule. Every liked video is a bookmark. The GitHub Pages site is the screen in my living room, and GitHub Actions is the signal tower making sure the schedule is always up to date.

Before building any feature, this vision must be clear. Everything that follows — every workflow, every page template, every line of code — serves this aspiration.
